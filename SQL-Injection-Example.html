Security Insight: SQL Injection

SQL Injection occurs when an attacker inserts malicious SQL into an application's inputs to manipulate or control the database.

Why It Happens

  • Unsanitised user input
  • String-built SQL queries
  • No input validation
  • Lack of parameterised queries

How to Prevent It

  • Use prepared statements
  • Validate and sanitise input
  • Use least-privilege DB accounts
  • Avoid string concatenation in SQL

2. Introduction to SQL Injection

SQL Injection allows attackers to modify database queries by injecting malicious SQL into application inputs.

Consequences include:

  • Authentication bypass
  • Accessing or deleting data
  • Privilege escalation
  • Full system compromise

SQL Injection remains one of the most common and critical vulnerabilities (OWASP Top 10 ‚Äì Injection).

3. Why SQL Injection Happens

5. Demo Scenario (SQL Injection in Action)

5.1 Intended Query

SELECT * FROM users
WHERE username = 'USER_INPUT'
AND password = 'PASS_INPUT';

5.2 Example Input

Username: admin' --
Password: anything

5.3 Resulting Malicious Query

SELECT * FROM users
WHERE username = 'admin' --'
AND password = 'anything';

The -- comment truncates the password check, bypassing authentication.

7. How to Prevent SQL Injection

  • Use parameterised queries (prepared statements)
  • Validate and sanitise input (whitelisting expected formats)
  • Use least‚Äëprivilege database accounts
  • Avoid dynamic SQL where possible
  • Use a Web Application Firewall (WAF) for common payloads
